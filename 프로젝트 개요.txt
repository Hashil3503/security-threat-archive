프로젝트 개요 (Project Overview)
프로젝트명: Security Threat Archive (보안 위협 아카이브)

주제: 다양한 네트워크 보안 위협 사례와 데이터 침해 기법을 기록하고 분석하는 웹 기반 대시보드 시스템입니다.

목표:

서비스 거부 공격(DoS, Ping of Death), 데이터 탈취(Sniffing, Session Hijacking), 소셜 엔지니어링(Smishing) 등 다양한 보안 위협을 체계적으로 분류하여 저장합니다.

웹 인터페이스를 통해 직관적으로 데이터를 조회하고 관리(CRUD)하는 전체 흐름을 완성합니다.

기술 스택 (Tech Stack)
우리가 구성한 완벽한 3계층(3-Tier) 아키텍처 기술들입니다.

프론트엔드 (Front-End): HTML5, CSS3, JavaScript (Vanilla JS, Fetch API)

백엔드 (Back-End): Java 17, Spring Boot 3.x (Spring Web, Spring Data JPA)

데이터베이스 (Database): MariaDB

빌드 툴 및 개발 환경: Gradle, Eclipse (STS5 Plugin)

상세 요구사항 및 개발 목표 (Requirements)
단순히 기능을 만드는 것을 넘어, 핵심 IT 기술을 실전에서 어떻게 활용하는지 경험하는 것에 초점을 맞춥니다.

1. 데이터베이스 요구사항 (DB & SQL)
DDL 활용: 데이터 정의어(CREATE, ALTER 등)를 사용하여 위협 카테고리 테이블과 상세 로그 테이블을 설계하고 구조화합니다.

DML 및 제어 활용: 로그 데이터를 실질적으로 삽입, 수정, 삭제하는 작업을 수행합니다. 이 과정에서 데이터의 무결성을 유지하고, 시스템의 안정성을 확보하는 권한 제어(DCL)의 기본 개념을 체감할 수 있도록 구성합니다.

관계형 데이터 조합: 카테고리(분류) 테이블과 위협 로그 테이블을 분리한 뒤, JOIN 연산을 활용해 흩어진 데이터를 하나의 표로 완벽하게 조합하여 대시보드에 출력합니다.

2. 백엔드 요구사항 (Java & Spring)
REST API 서버 구축: 프론트엔드의 요청(데이터 저장, 조회, 삭제)을 받아 데이터베이스로 넘겨주는 다리 역할을 구현합니다.

객체 지향 원칙 적용: 컨트롤러, 서비스, 데이터 접근(Repository) 계층을 명확히 분리합니다. 또한 비즈니스 로직을 처리할 때 클래스 상속이나 메서드 오버라이딩과 같은 Java의 핵심 구조를 활용하여 코드의 중복을 줄이고 재사용성을 높이는 연습을 진행합니다.

3. 프론트엔드 요구사항 (UI/UX)
비동기 통신 구현: 화면 전체를 새로고침하지 않고도, JavaScript를 이용해 백엔드 서버와 데이터를 매끄럽게 주고받아 화면을 동적으로 업데이트합니다.

직관적인 시각화: 위협의 심각도(High, Medium, Low)에 따라 시각적인 강조 효과(예: 위험 데이터는 붉은색)를 CSS로 구현하여 가독성과 사용성을 높입니다.

이제 이 명확한 청사진을 바탕으로, 이클립스에 설치한 STS5 플러그인을 활용하여 프로젝트 뼈대를 만들 차례입니다.
